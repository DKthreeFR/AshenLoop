## 2. 环境与问题排查结论

### 2.1 PowerShell 问题根因

- 报错来自 `D:\annacoda3\shell\condabin\Conda.psm1:76` 的 `Invoke-Expression`。
- 根因是系统级 `PATH` 包含非法条目：
- `"D:\apache-tomcat-10.0.23\bin;D:\apache-tomcat-10.0.23\lib"`
- 此条目带引号且内部含分号，导致 conda 生成的激活脚本被 PowerShell 解析失败。

### 2.2 修复建议

- 删除错误 PATH 条目。
- 改为两个独立条目：
- `D:\apache-tomcat-10.0.23\bin`
- `D:\apache-tomcat-10.0.23\lib`
- 重新打开 PowerShell 生效。

## 3. 角色控制器设计结论

### 3.1 架构方向

- 采用“分层状态机（HFSM） + 模块化分层 + 数据驱动”。
- 强调逻辑与表现分离：
- 代码状态机（业务规则）为主。
- Animator（动画表现）为从。

### 3.2 面试表达要点

- 复杂度小可先用 FSM，状态增长后演进到 HFSM。
- HFSM 优势：减少状态爆炸、复用父层规则、扩展维护成本低。

### 3.3 Animation Sync 关键认知

- Animation Sync 是“逻辑状态 -> Animator 参数”的翻译层。
- 输入不直接调用动画切换。
- 正确链路：
- `Input -> StateMachine -> OnStateChanged -> AnimationSync -> Animator`
