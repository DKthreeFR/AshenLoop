## 新增能力与功能结果

1. 背包新增“消耗物品”流程  
   在 `InventoryManager` 中新增 `TryConsumeItem(Item item, int count)`，可以扣减指定物品数量，数量耗尽后清空对应槽位，并执行 `RefreshItem + SaveBag`。

2. 背包面板接入“使用按钮”逻辑  
   `BagPanelController` 新增 `BtnUseClicked` 绑定，在点击使用时根据当前选中物品触发消耗流程并执行玩家属性变化（示例为加血）。

3. 玩家运行时模型与存档体系落地  
   新增：
   - `GameManager` 单例（跨场景常驻）
   - `PlayerModel`
   - `PlayerSaveData`
   - `PlayerSaveSystem`

   形成了玩家属性的统一入口与持久化能力。

4. 游戏面板接入玩家生命值显示  
   `GamePanelController` 注入并订阅 `PlayerModel.OnStatsChanged`，`GamePanelView` 新增 `HpValueFillBar` 与更新方法，用于实时刷新血条。

5. 背包槽位选中信息联动  
   `SingleSlot` 补充了选中物品信息传递能力（包含对 `GameManager.Instance.playerModel.selectedItem` 的写入），为“点击物品后使用”提供数据来源。

## 本次实现的流程闭环

`点击背包槽位 -> 记录 selectedItem -> 点击 BtnUse -> TryConsumeItem 扣减数量 -> RefreshItem 刷 UI -> SaveBag 持久化`  
并且在玩家属性侧形成：  
`使用物品 -> PlayerModel 属性变化 -> OnStatsChanged -> GamePanel 血条更新`

## 当前已知风险/后续建议

1. `PlayerModel` 中每次 `SetXxx` 都执行 `OnStatsChanged += SavePlayerModel`，会重复注册同一回调，后续应改为构造时注册一次或直接在 `SetXxx` 内显式调用保存函数。

2. `GamePanelView.UpdateHpValueFillBar()` 当前未做 `MaxHp<=0` 保护，建议使用 `Mathf.Clamp01(hp / maxHp)` 并防分母为 0。

3. `BagPanelController.OnBtnUseClick()` 直接访问 `selectedItem.itemID`，如果未选中物品会触发空引用，建议先判空再分支。
