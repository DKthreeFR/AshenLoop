## 三、功能增量（详细）

### 1) 任务系统“业务层”独立化

新增目录 Task/，任务系统从 UI 脚本中拆出来，形成独立职责层：

1. TaskData.cs：任务定义（SO 配置）
- 保存 	askId / taskName / desc / targetType / targetCount / rewardGold

2. TaskStatus.cs：任务生命周期
- Locked / Available / InProgress / Completed / Submitted

3. TaskRunTime.cs：任务运行态
- 记录当前进度、状态、追踪标记、时间戳
- 提供 ApplyProgress / MarkAccepted / MarkSubmitted

4. TaskService.cs：任务核心逻辑
- 初始化定义字典
- 接取任务（动态创建 runtime）
- 推进进度
- 提交任务
- 构建 UI 快照 BuildPanelItems()
- 通过 TasksChanged 事件通知 UI 刷新

5. TaskManager.cs：统一入口转发
- 对外暴露 AcceptTask 和 NotifyTaskProgress
- 内部转发到 TaskService

结果：任务业务与 UI 表现解耦，后续可继续扩展存档、前置任务链、奖励发放。

### 2) 动态接任务流程落地

TaskService.Initialize(...) 中默认创建 runtime 的逻辑被去除（注释掉），改为：
- 只加载任务定义
- 在 AcceptTask(taskId) 发生时才创建 runtime 并标记进行中

结果：任务面板不再“开局显示全部定义任务”，而是“接取后才显示”。

### 3) TaskPanel MVC 联动增强

1. TaskPanelController 重建
- 统一订阅 View / Model / Service 三类事件
- OnTasksChanged -> Model.SetTasks(...)
- OnModelDataChanged -> View.RenderTaskList(...)
- OnModelSelectedChanged -> View.RenderTaskList(...) + RefreshDetail()
- 增加 SubmitClicked 处理，支持提交任务

2. TaskPanelModel 增强
- 新增 SelectedTaskId
- 新增 OnSelectedChanged
- SetTasks 支持默认选中第一条
- 新增 TryGetSelectedItem

3. TaskPanelView 增强
- 支持动态渲染左侧任务按钮列表
- 支持右侧详情显示（名称/描述/进度/状态）
- 支持提交按钮可交互状态
- 支持任务项点击事件 TaskItemClicked

结果：任务面板从“静态按钮”升级为“数据驱动列表 + 选中详情联动”。

### 4) GameMain 接线完善

Main/GameMain.cs 中新增任务系统接线：
- 创建并初始化 TaskService
- 注入 TaskPanelController.SetTaskService(_taskService)
- 注入 TaskManager.Instance.SetTaskService(_taskService)
- 并保持 UIManager 的 TaskPanel 绑定

此外加入了示例调用：
- TaskManager.Instance.AcceptTask("2")

结果：场景启动后任务系统有完整依赖链，不再依赖 UI 脚本内部硬编码状态。

## 五、当前风险与建议

1. 状态文案映射仍偏简化
- 目前详情状态主要通过 CanSubmit 推导，建议后续把 TaskStatus 映射到 UI（如“已提交/进行中/可提交”）避免语义偏差。

2. 代码里存在重复渲染路径风险
- TaskPanelView 同时存在 RenderTaskList 与 RebuildTaskButtons（尽管你已在 Controller 注释其中一条），建议保留一个入口。

3. taskId 字符串依赖强
- AcceptTask("2") 完全依赖 SO 中 	askId 一致性，建议后续用常量或配置校验工具避免拼写错误。

4. 注释编码乱码
- 多个文件注释存在乱码，建议统一 UTF-8 编码，降低维护负担。
