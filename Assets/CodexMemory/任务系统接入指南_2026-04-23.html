Step C：TaskPanelController 退订补全

protected override void OnDispose()

4. 如何实现“左侧按钮切换右侧详情”（你最关心）

你要新增的最小字段

在 TaskPanelModel.cs 里增加：

public string SelectedTaskId { get; private set; }
public event Action OnSelectedChanged;

public void SelectTask(string taskId)
{
    if (SelectedTaskId == taskId) return;
    SelectedTaskId = taskId;
    OnSelectedChanged?.Invoke();
}

Controller 做两件事

  1. 收到 TasksChanged 后：Model.SetTasks(service.BuildPanelItems())
  2. 收到 View 某个任务按钮点击后：Model.SelectTask(taskId)

View 渲染逻辑（关键）

  1. 左侧：根据 Model.Items 动态生成按钮
  2. 点击按钮：抛出 TaskItemClicked(taskId)
  3. 右侧：根据 Model.SelectedTaskId 找到那条数据并显示详情

默认选中策略：首次打开面板，如果 SelectedTaskId 为空，就自动选第一条任务。

5. 运行时怎么驱动任务进度

在任意游戏逻辑里（怪物死亡、采集成功、交互完成）调用：

TaskManager.Instance.NotifyTaskProgress("Kill_Wolf", 1);
TaskManager.Instance.NotifyTaskProgress("Collect_Herb", 2);

然后链路是：

TaskManager.NotifyTaskProgress
    -> TaskService.NotifyTaskProgress
    -> TasksChanged 事件
    -> TaskPanelController.OnTasksChanged
    -> Model.SetTasks(...)
    -> View 重绘左侧列表 + 右侧详情

6. 你现在项目里最容易踩坑的点

  • TaskService 未注入：UI 有类但不会动（你当前就是这个状态）。
  • Controller 没退订事件：切场景后重复刷新、空引用。
  • TaskData 的 targetType 用字符串：后续建议改 enum 或常量表，避免拼写错误。
  • 注释编码乱码：建议统一 UTF-8，否则维护成本会越来越高。
