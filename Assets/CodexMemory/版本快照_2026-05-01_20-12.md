## 本次已修改文件

### 1. Monster/Monster.cs

文件：
- `Assets/Scripts/Monster/Monster.cs`

改动：
- 增加 `using System;`
- 将泛型参数 `T value` 的强制转换从 `(float)value` 改为 `Convert.ToSingle(value)`

原因：
- `SendEvent<T>(string eventName, T value)` 中的 `value` 是泛型，编译器不能保证 `T` 一定是 `float`。
- `(float)value` 在泛型上下文中不安全，容易编译失败。
- `Convert.ToSingle(value)` 可以兼容 `int`、`float`、`double` 等常见数值类型。

当前逻辑：
```csharp
if(eventName == "PlayerAttackMonster")
{
  MonsterHpUpdate(Convert.ToSingle(value));
}
```

风险：
- 如果传入的 `value` 不是数值类型，运行时仍会抛转换异常。
- 后续可以根据需要改成 `try/catch` 或类型模式匹配，避免非数值输入导致报错。

### 玩家异步加载后，攻击键和跳跃键失效

用户现象：
- 玩家通过异步加载实例化到场景后：
  - 行走、跑步正常。
  - 攻击键、跳跃键无效。

重点检查文件：
- `Assets/Scripts/CharacterController/Input/CharacterControlDriver.cs`
- `Assets/Scripts/CharacterController/Input/CharacterInputAdapter.cs`
- `Assets/Scripts/CharacterController/Input/CharacterInputBuffer.cs`
- `Assets/Scripts/CharacterController/Input/CharacterInputCommand.cs`
- `Assets/Scripts/CharacterController/Input/CharacterInputCommandProvider.cs`

- `Assets/Scripts/Main/GameMain.cs`
- `Assets/Scripts/AAMgr.cs`
- `Assets/Scripts/CharacterController/StateMachine/CharacterStateControl.cs`
- `Assets/Scripts/CharacterController/StateMachine/AttackState.cs`
- `Assets/Scripts/CharacterController/StateMachine/JumpState.cs`
- `Assets/Scripts/CharacterController/StateMachine/MoveState.cs`
- `Assets/MyInputSystem/GameInputAction.inputactions`
- `Assets/ArtRes/Prefab/Player/Player.prefab`

已确认输入链路：
```text
CharacterInputAdapter
  -> CharacterInputCommandProvider
  -> PlayerInputSource
  -> CharacterInputBuffer
  -> CharacterControlDriver
  -> CharacterStateControl
  -> AttackState / JumpState
```

关键结论：
- 移动能生效，是因为 `MoveInput` 是持续状态，每帧从 `CharacterInputAdapter.MoveInput` 同步。
- 攻击和跳跃不同，它们是一次性事件：
  - `CharacterInputAdapter.AttackPressed`
  - `CharacterInputAdapter.JumpPressed`
  - 再写入 `CharacterInputCommand.AttackPressed / JumpPressed`
- 如果 Adapter 事件没有触发，状态机后面完全不会进入 Attack 或 Jump。

`.inputactions` 检查结果：
- `Attack` 在 `Move` action map 中，绑定 `<Keyboard>/j`
- `Jump` 在 `Move` action map 中，绑定 `<Keyboard>/space`
- `DodgeRun` 绑定 `<Keyboard>/shift`
- `Move` 使用 WASD

预制体检查结果：
- `Assets/ArtRes/Prefab/Player/Player.prefab` 上存在输入链组件。
- `CharacterControlDriver.inputSourceBehaviour` 指向 `CharacterInputBuffer`。
- `CharacterInputBuffer.inputSourceBehaviour` 指向 `PlayerInputSource`。
- `PlayerInputSource` 和 `CharacterInputCommandProvider` 都依赖自动查找同物体组件。

目前最可疑点：
1. `CharacterInputAdapter` 的 Attack / Jump 回调是否真的触发。
2. Addressables 异步加载出来的 Player 是否使用了最新 Prefab 和最新输入资源。
3. `CharacterInputCommandProvider.OnEnable()` 执行时，`inputAdapter` 是否已正确找到并完成事件订阅。
4. `Jump` 状态有进入条件限制：只有当前状态是 `Move/Run/FightMove/FightRun` 时才允许进入。
5. `Attack` 状态依赖 `weaponComboConfigs[0]`、Animator 参数和 `CharacterEvent`，如果这些引用在异步实例中缺失，可能进入后立刻退出或看起来没有表现。

建议下一步验证：
1. 在异步实例化出的 Player 上挂或启用 `Test.cs`，观察按 `J` 和 `Space` 时是否打印 Attack / Jump 日志。
2. 如果 `Test.cs` 没有日志，问题在 `CharacterInputAdapter` 或 InputAction 配置/启用。
3. 如果 `Test.cs` 有日志，但状态机不切状态，问题在 `CharacterInputCommandProvider -> Buffer -> Driver -> CharacterStateControl` 之间。
4. 如果状态机已切 Attack / Jump，但动画没表现，问题在 Animator 参数、动画事件、武器配置或 CharacterEvent 引用。
