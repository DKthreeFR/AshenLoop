# 角色控制器设计文档模板（Unity ARPG）

## 1. 文档目的

本模板用于指导你搭建一个高质量、易维护、可扩展的角色控制器系统。  
核心思路：分层解耦（输入/状态/运动/战斗/表现/数据），并通过统一命名与清晰目录降低维护成本。

## 2. 设计目标

- 高质量手感：移动、翻滚、攻击、受击行为稳定。
- 易维护：每个脚本只负责单一领域，避免 God Object。
- 易扩展：后续新增技能、武器、Buff 时不推翻现有结构。
- 可测试：关键行为可通过调试面板与状态日志验证。

## 3. 模块划分（大图）

- Core：总控、上下文、模块生命周期。
- Input：输入来源统一为命令。
- StateMachine：状态切换与动作锁/打断规则。
- Movement：位移、朝向、地面检测。
- Combat：攻击流程、命中检测、伤害结算。
- Animation：Animator 参数同步与动画事件中继。
- Stats：属性与资源（HP/Stamina 等）。
- Interaction：交互（拾取、对话、开箱等）。
- Data：运行时数据与存档映射。
- Debug：可视化调试与运行诊断。
- Editor：编辑器检查与配置工具。

## 4. 建议目录结构

```text
Assets/Scripts/CharacterController/
├─ Core/
├─ Input/
├─ StateMachine/
├─ Movement/
├─ Combat/
├─ Animation/
├─ Stats/
├─ Interaction/
├─ Data/
├─ Debug/

```

## 5. 文件清单与职责映射（重点）

### 5.1 Core（系统骨架）

- `Core/CharacterControllerFacade.cs`  
  对应部分：系统总入口（模块初始化、调度 Update/FixedUpdate、模块间连接）。
- `Core/CharacterContext.cs`  
  对应部分：共享上下文（缓存组件引用、运行时共享数据）。
- `Core/CharacterEvents.cs`  
  对应部分：全局事件中心（如 OnAttackStart/OnHit/OnDeath）。
- `Core/ICharacterModule.cs`  
  对应部分：模块生命周期统一接口（Initialize/Tick/Dispose）。
- `Core/TickGroup.cs`  
  对应部分：更新分组策略（Update/FixedUpdate/LateUpdate 的执行归类）。

### 5.2 Input（输入层）

- `Input/CharacterInputAdapter.cs`  
  对应部分：将外部输入映射为统一命令。
- `Input/CharacterInputCommand.cs`  
  对应部分：输入命令定义（Move/Attack/Dodge/Skill/Interact）。
- `Input/CharacterInputBuffer.cs`  
  对应部分：输入缓冲（连段窗口、提前输入容错）。
- `Input/ICharacterInputSource.cs`  
  对应部分：输入来源抽象接口。
- `Input/PlayerInputSource.cs`  
  对应部分：玩家输入实现。
- `Input/AIInputSource.cs`  
  对应部分：AI 驱动输入实现（后续可接行为树）。

### 5.3 StateMachine（行为编排）

- `StateMachine/CharacterStateMachine.cs`  
  对应部分：状态机核心（当前状态、切换、条件评估）。
- `StateMachine/CharacterStateId.cs`  
  对应部分：状态枚举与索引。
- `StateMachine/ICharacterState.cs`  
  对应部分：状态接口（Enter/Update/Exit）。
- `StateMachine/StateTransitionRule.cs`  
  对应部分：状态切换规则与优先级。
- `StateMachine/States_Locomotion_Idle.cs`  
  对应部分：站立状态逻辑。
- `StateMachine/States_Locomotion_Move.cs`  
  对应部分：移动状态逻辑。
- `StateMachine/States_Locomotion_Dodge.cs`  
  对应部分：闪避/翻滚状态。
- `StateMachine/States_Combat_Attack.cs`  
  对应部分：攻击状态（连段窗口、动作锁）。
- `StateMachine/States_Reaction_Hit.cs`  
  对应部分：受击状态（硬直、击退）。
- `StateMachine/States_Global_Dead.cs`  
  对应部分：死亡状态（禁输入、禁切换）。

### 5.4 Movement（运动层）

- `Movement/CharacterMotor.cs`  
  对应部分：底层位移计算（速度/加速度/重力）。
- `Movement/CharacterMovementController.cs`  
  对应部分：高层移动决策（普通移动、冲刺、翻滚请求）。
- `Movement/GroundChecker.cs`  
  对应部分：地面检测与落地判定。
- `Movement/RotationController.cs`  
  对应部分：角色朝向控制（输入朝向/目标锁定朝向）。
- `Movement/MovementConfigSO.cs`  
  对应部分：移动参数配置（最大速度、加速度、转向速度等）。

### 5.5 Combat（战斗层）

- `Combat/CharacterCombatController.cs`  
  对应部分：战斗请求入口与攻击流程总控。
- `Combat/AttackRequest.cs`  
  对应部分：攻击请求数据结构（武器、技能、方向、时机）。
- `Combat/AttackDefinitionSO.cs`  
  对应部分：攻击配置（前摇/生效帧/后摇/打断性）。
- `Combat/HitboxEmitter.cs`  
  对应部分：命中盒发射（攻击方）。
- `Combat/HurtboxReceiver.cs`  
  对应部分：受击盒接收（被攻击方）。
- `Combat/DamageService.cs`  
  对应部分：伤害计算入口（统一调用）。
- `Combat/DamageFormula.cs`  
  对应部分：伤害公式实现（ATK/DEF/暴击等）。
- `Combat/InterruptPolicy.cs`  
  对应部分：打断规则（可打断/不可打断、霸体、硬直）。

### 5.6 Animation（表现同步层）

- `Animation/CharacterAnimationSync.cs`  
  对应部分：逻辑状态到 Animator 参数同步。
- `Animation/AnimatorParamIds.cs`  
  对应部分：Animator 参数常量/Hash 统一管理。
- `Animation/AnimationEventRelay.cs`  
  对应部分：动画事件转发（出手帧、无敌帧开始/结束）。
- `Animation/AnimationStateTags.cs`  
  对应部分：动画状态标签定义（Locomotion/Attack/Hit/Dead）。

### 5.7 Stats（属性资源层）

- `Stats/CharacterStats.cs`  
  对应部分：角色基础属性与最终属性容器。
- `Stats/CharacterResources.cs`  
  对应部分：资源值（HP/Stamina/Mana）管理。
- `Stats/StatModifier.cs`  
  对应部分：属性修饰器（装备、Buff、临时效果）。
- `Stats/CharacterStatsConfigSO.cs`  
  对应部分：基础属性配置。
- `Stats/StatRecalculationService.cs`  
  对应部分：属性重算服务（变更后统一重算）。

### 5.8 Interaction（交互层）

- `Interaction/CharacterInteractionController.cs`  
  对应部分：交互检测与触发入口。
- `Interaction/IInteractable.cs`  
  对应部分：可交互对象协议。
- `Interaction/InteractionQuery.cs`  
  对应部分：交互目标筛选规则（距离、角度、优先级）。
- `Interaction/InteractionConfigSO.cs`  
  对应部分：交互参数配置（范围、层级、冷却）。

### 5.9 Data（数据层）

- `Data/CharacterRuntimeData.cs`  
  对应部分：运行时临时数据快照。
- `Data/CharacterSaveData.cs`  
  对应部分：可序列化存档结构。
- `Data/CharacterLoadDataMapper.cs`  
  对应部分：存档数据与运行时数据的双向映射。

### 5.10 Debug（调试层）

- `Debug/CharacterDebugPanel.cs`  
  对应部分：运行时调试面板（状态、速度、输入、冷却）。
- `Debug/CharacterDebugGizmos.cs`  
  对应部分：场景可视化（检测范围、命中盒、朝向）。
- `Debug/CharacterDebugOptionsSO.cs`  
  对应部分：调试开关配置。

### 5.11 Editor（编辑器工具）

- `Editor/CharacterControllerValidator.cs`  
  对应部分：配置校验（缺组件、空配置、非法参数）。
- `Editor/CharacterConfigInspector.cs`  
  对应部分：自定义 Inspector，提升配置效率。

## 6. 命名规范

- 类名：`PascalCase`，统一前缀 `Character`（如 `CharacterMotor`）。
- 接口：`I` 前缀（如 `ICharacterState`）。
- ScriptableObject：`SO` 后缀（如 `MovementConfigSO`）。
- 状态文件：`States_域_状态.cs`（便于检索）。
- 服务类：`...Service.cs`（只处理逻辑，不挂表现）。

## 7. 创建顺序（建议）

1. Core + Input（打通基础数据流）
2. StateMachine（先跑 Idle/Move/Dodge）
3. Movement（先拿到稳定移动手感）
4. Combat（普攻闭环、受击打断）
5. Animation（参数同步与事件中继）
6. Stats + Interaction
7. Data（存档/读档）
8. Debug + Editor（提升迭代效率）

## 8. 最小可运行里程碑（MVP）

- 角色可移动、可翻滚、可普攻连段。
- 命中后触发受击与伤害结算。
- 动画参数与逻辑状态一致。
- 调试面板可实时看到：状态、输入、速度、HP、Stamina。

## 9. 验收检查清单

- 状态切换是否可预测且可复现。
- 关键参数是否全部配置化（非硬编码）。
- 战斗逻辑是否集中在 Combat 层。
- Animator 是否仅做表现同步，不承载业务规则。
- 新增一个技能时，是否只需改配置和少量模块。




• 当前这套角色控制器设计覆盖的功能是：

    1. 输入统一接入

  - 玩家输入和 AI 输入都可映射为统一命令（移动、攻击、闪避、交互）。

    2. 分层状态管理

  - 顶层状态：Locomotion / Combat / Hit / Dead。
  - 子状态：Idle / Move / Dodge / Attack1/2...。
  - 支持动作锁、打断、恢复。

    3. 角色移动能力

  - 位移、转向、地面检测、重力处理。
  - 支持冲刺/翻滚这类高机动动作。

    4. 基础战斗闭环

  - 攻击请求 -> 攻击状态 -> 命中检测（Hitbox/Hurtbox）-> 伤害结算。
  - 支持连段窗口与受击打断规则。

    5. 动画同步机制

  - 逻辑状态同步到 Animator 参数。
  - 动画事件回传逻辑层（如出手帧、无敌帧开始/结束）。

    6. 属性与资源系统

  - 管理 HP、Stamina、攻击/防御等属性。
  - 装备/Buff 导致属性变化时可统一重算。

    7. 交互能力

  - 可交互对象检测与触发（拾取、开箱、对话等）。

    8. 数据与调试支持

  - 运行时数据与存档结构预留。
  - 调试面板/Gizmos 便于排查状态与手感问题。

  简化理解：它已经是一个“可跑 ARPG 核心循环”的控制器骨架，后续只需往技能、武器、UI、存档深挖。
