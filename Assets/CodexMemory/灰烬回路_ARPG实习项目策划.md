# 《灰烬回路：锈城试炼》ARPG实习项目策划（Unity）

> 面向对象：`27届`大三、Unity 约半年经验、目标是暑期游戏客户端实习  
> 核心策略：`做得完 > 做得大`，用中小体量做出“系统完整度 + 工程规范 + 可展示性”

## 1. 项目定位

- 项目类型：俯视角（Top-Down）单机 ARPG
- 引擎版本：Unity `2022 LTS` 或 `2021 LTS`
- 目标时长：`4~6 周`做出可演示版本（MVP+）
- 核心卖点：
- 手感扎实的近战战斗（普攻 + 技能 + 闪避）
- 完整的RPG外围系统（背包、装备、掉落、属性、存档）
- 清晰的UI信息架构（战斗HUD + 背包页 + 角色页）

## 2. 一句话玩法

玩家在“锈城遗迹”中清理怪物、收集装备、强化角色，挑战关底守卫。  
核心循环：`探索 -> 战斗 -> 掉落 -> 配装 -> 变强 -> 挑战更强敌人`

## 3. Scope控制（非常关键）

## 必做（MVP）

- 一个可反复刷的主关卡 + 一个Boss房
- 玩家基础移动、普攻连段（2~3段）、闪避无敌帧
- 2个主动技能（一个单体爆发、一个范围清怪）
- 3类小怪 + 1个Boss（基础AI）
- 装备系统：武器/护甲/饰品 三槽
- 背包系统：格子背包、拾取、使用、丢弃、装备/卸下
- 掉落系统：怪物掉落概率 + 品质（普通/稀有/史诗）
- 属性系统：攻击、防御、生命、暴击（够用即可）
- UI系统：战斗HUD、背包页、角色页、暂停菜单
- 存档系统：本地JSON（角色等级、装备、背包、关卡进度）

## 可选加分（时间有余再做）

- 任务系统（1条主线 + 2条支线）
- 装备词条随机（如 +5%暴击）
- 技能树（2选1分支）
- 小地图、伤害飘字、命中停顿

## 不做（避免爆炸）

- 联机
- 开放世界
- 复杂剧情演出
- 多职业并行开发
- 大规模程序化地图

## 4. 核心系统设计

## 4.1 战斗系统

### 玩家控制

- 移动：`WASD`（或左摇杆）
- 朝向：鼠标/右摇杆方向
- 普攻：近战扇形检测（Box/Circle Cast）
- 闪避：短位移 + 无敌帧 `0.2~0.35s` + CD `2.5s`

### 技能设计（2个就够）

- 技能A【裂钢斩】：
- 前方短距离高伤单体，CD `4s`
- 用于打精英/Boss破绽
- 技能B【余烬震荡】：
- 角色周围圆形AOE，中伤害+击退，CD `8s`
- 用于清怪和解围

### 受击反馈（手感分）

- 命中停顿 `0.03~0.06s`
- 受击闪白（材质参数）
- 屏幕轻微震动（Cinemachine Impulse）
- 伤害数字浮动（对象池）

## 4.2 敌人与AI

### 敌人类型

- 近战杂兵：追击+贴身攻击
- 远程投射：保持距离，定期射击
- 冲锋怪：蓄力后直线冲刺
- Boss：三段行为循环（追击/范围技/召唤）

### AI状态机（简化）

- `Idle -> Patrol -> Chase -> Attack -> Dead`
- Boss可追加：`Phase2`（血量<50%时技能变化）

## 4.3 属性与成长

- 基础属性：
- `HP`、`ATK`、`DEF`、`CRIT`
- 计算简式（可读性强）：
- 最终伤害 = `max(1, 攻击方ATK * 技能倍率 - 防守方DEF)`
- 暴击 = `最终伤害 * 1.5`
- 成长：
- 击杀获得经验，升1~5级即可（够演示）
- 每级 + 固定属性（不要做复杂加点）

## 4.4 装备与掉落

### 装备槽位

- 武器（影响ATK）
- 护甲（影响DEF、HP）
- 饰品（影响CRIT 或技能CD）

### 品质系统

- 普通（白）
- 稀有（蓝）
- 史诗（紫）

### 掉落规则（示例）

- 小怪：白70% / 蓝25% / 紫5%
- 精英：白40% / 蓝45% / 紫15%
- Boss：蓝60% / 紫40%

## 4.5 背包系统

### 数据结构建议

- `ItemData`（ScriptableObject）：静态配置（ID、名称、图标、描述、品质、类型、基础数值）
- `ItemInstance`（C# class）：动态实例（唯一ID、词条、数量、耐久可选）

### 功能清单

- 拾取到背包空格
- 相同消耗品叠加（如药水上限99）
- 点击查看详情
- 装备/卸下
- 使用（药水回复）
- 丢弃（生成地面掉落物）
- 背包满提示（Toast）

### UI交互细节

- 左键：选中/拖拽
- 右键：快捷使用或装备
- Tooltip显示：名称、品质色、属性变化（绿色+、红色-）

## 4.6 UI系统设计

### 页面结构

- 战斗HUD：
- 血条、蓝条/能量、技能CD、快捷栏、经验条、当前任务
- 背包页：
- 物品网格 + 详情面板 + 操作按钮
- 角色页：
- 当前装备、总属性、战力值（可选）
- 暂停页：
- 继续、设置、返回主菜单、退出

### UI技术方案

- 使用 `UGUI` 即可（不强制UI Toolkit）
- UI层级：
- `HUD Canvas`
- `Menu Canvas`
- `Popup Canvas`
- 统一UI管理：
- `UIManager`负责打开/关闭页面
- 每个页面继承 `UIPanelBase`

## 4.7 存档系统

- 存档内容：
- 玩家等级、经验、当前位置（或关卡ID）
- 背包物品列表（ItemInstance序列化）
- 已装备物品
- 关键设置（音量、画质可选）
- 技术实现：
- `SaveData` C#结构 + `JsonUtility`/`Newtonsoft.Json`
- 本地路径：`Application.persistentDataPath`

## 5. 程序架构建议（实习面试会问）

## 推荐目录

```text
Assets/
  Art/
  Audio/
  Prefabs/
  Scenes/
  Scripts/
    Core/           # GameEntry, EventBus, ObjectPool
    Character/      # Player, Enemy, Stats
    Combat/         # Damage, Hitbox, Skill
    Inventory/      # Item, Bag, Equipment
    UI/             # UIManager, Panels, Widgets
    SaveLoad/       # SaveSystem, SaveData
```

## 核心设计原则

- 配置与逻辑分离：数值放 `ScriptableObject`
- 能事件就事件：减少硬引用（如 UI刷新用事件）
- 对象池管理频繁对象（子弹、飘字、掉落）
- 单例适度使用：`GameManager/UIManager`够了，不要全单例

## 6. 场景与关卡设计

- 主场景：锈城外围（教学+刷怪）
- 中段：遗迹走廊（资源补给点）
- 终段：核心熔炉（Boss战）
- 节奏：
- 每 `2~3`波战斗给一次恢复机会
- 关底前提供一个“装备整理点”

## 9. 风险清单与兜底策略

- 风险：战斗手感调不出来
- 兜底：先做“能打中 + 有反馈”，再迭代细节
- 风险：背包与装备数据耦合混乱
- 兜底：先定义`ItemData/ItemInstance`再写UI
- 风险：UI耗时过长
- 兜底：先功能完成，最后统一美术皮肤
- 风险：Boss做太复杂
- 兜底：先做3招循环，保证稳定可演示
