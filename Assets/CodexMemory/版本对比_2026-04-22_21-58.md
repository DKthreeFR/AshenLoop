## 三、本次新增能力（功能视角）

### 1) 场景加载流程支持 Started 回调
AAMgr.LoadSceneAsync(...) 新增 onStarted 回调参数，在协程启动加载前触发。

落地效果：
- 可以在真正加载前立即打开 LoadingPanel
- 避免“首帧进度已开始但 UI 还没打开”的时序问题

### 2) UIManager 强化为可跨场景复用的幂等管理器
UIManager 本次补了两类关键能力：

1. 单例防重复实例
- Awake 中新增重复实例检查并销毁
- 保留唯一常驻实例（DontDestroyOnLoad）

2. 幂等绑定
- 新增 IsControllerBound(UIPanelId)
- BindController 增加“已绑定即跳过初始化（按需 Show）”逻辑

落地效果：
- 重复进入场景不再反复初始化同一 Controller
- 避免多次绑定导致事件重复订阅

### 3) UI 架构从“单面板演示”扩展到多面板 MVC
新增了 Main/Game/Loading/Setting 四组 Panel 的 Model/View/Controller 脚本骨架。

落地效果：
- 主菜单、游戏内、加载、设置面板有了独立职责
- 后续功能扩展（按钮、文本、动画）可直接在对应 Controller 上迭代

### 4) 场景入口拆分
原先根目录 Main.cs 删除，改为：
- Main/Main.cs：主菜单场景入口
- Main/GameMain.cs：游戏场景入口

落地效果：
- 场景职责更清晰
- UI 绑定和鼠标锁定等流程不再混在一个入口脚本里

### 5) 主菜单到游戏场景的加载联动
MainPanelController 中：
- 点击开始后发起场景异步加载
- started 时打开 LoadingPanel
- progress 时广播进度（LoadingPanelModel.RaiseProgress）
- done 时关闭 MainMenu 与 LoadingPanel

落地效果：
- 主菜单进入游戏的加载链路打通

### 6) 游戏场景返回主菜单链路
GamePanelController 中：
- 退出按钮触发回主菜单场景加载
- 在 started 阶段先关闭 GamePanel

落地效果：
- 游戏内返回主菜单的基础流程已建立

### 7) 角色移动/跳跃方向一致性修正（进行中）
