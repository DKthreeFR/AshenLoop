## 新增功能点

### 1. 设置面板数据持久化

新增 `SettingPanelData`，用于保存：
- 音效开关：`isSoundOn`
- 音效音量：`soundVolume`
- 音乐开关：`isMusicOn`
- 音乐音量：`musicVolume`

`SettingPanelModel` 新增本地 JSON 读写：
- 存档路径：`Application.persistentDataPath/Setting.json`
- 如果文件存在则读取设置。
- 如果文件不存在则使用默认值：音效/音乐开启，音量为 `1f`。
- 关闭设置面板时通过 `SaveSetting()` 写入本地 JSON。

### 2. 设置面板 UI 事件接入

`SettingPanelView` 新增并绑定：
- 音效 Toggle
- 音效 Slider
- 音乐 Toggle
- 音乐 Slider
- 退出到主菜单按钮

`SettingPanelController` 新增事件处理：
- `OnSoundToggled`
- `OnSoundVolumeChanged`
- `OnMusicToggled`
- `OnMusicVolumeChanged`
- `OnExitToMainClicked`

### 3. 游戏内面板的退出按钮调整为设置入口

`GamePanelView` 将原来的 `BtnExitGame / ExitGameClicked` 改为：
- `BtnSettingGame`
- `SettingGameClicked`

`GamePanelController` 不再直接从游戏面板退出到主菜单，而是打开设置面板：

```text
GamePanel -> SettingPanel -> ExitToMain
```

这样退出游戏流程集中到设置面板中处理。

### 4. 主菜单设置面板复用 GameManager 中的 Model

`Main/Main.cs` 不再自己创建 `SettingPanelModel`。

现在绑定设置面板时使用：
- `GameManager.Instance.settingPanelModel`

影响：
- 设置数据入口更集中。
- 主菜单和游戏内设置面板更容易共享同一份设置数据。

### 5. 新增音频管理脚本雏形

新增 `AudioManager`：
- 获取当前物体上的 `AudioSource`
- `DontDestroyOnLoad(gameObject)` 保证跨场景保留

目前只是基础骨架，还没有接入设置面板的音量/开关事件。

### 6. 怪物 Canvas 初始隐藏

`Monster.OnEnable()` 中新增：
- `monsterCanvas.gameObject.SetActive(false);`

作用：
- 怪物启用时默认隐藏血条/Canvas，避免初始显示不符合预期。

### 7. 任务完成状态查询入口

`TaskManager` 新增：
- `CheckTaskStatus(string taskId)`

用途：
- 提供给行为树检查任务是否完成。

## 关键逻辑变更与影响范围

### 设置入口流程变化

原流程：
```text
GamePanel 点击退出 -> 直接 LoadSceneAsync(MainMenuScene)
```

新流程：
```text
GamePanel 点击设置 -> 打开 SettingPanel
SettingPanel 点击退出到主菜单 -> LoadSceneAsync(MainMenuScene)
```

影响：
- 游戏内退出行为从 GamePanel 迁移到 SettingPanel。
- GamePanel 更像 HUD 入口，SettingPanel 承担设置和退出流程。

### 设置数据生命周期变化

`SettingPanelModel` 从 Main 局部创建改为使用 `GameManager.Instance.settingPanelModel`。

影响：
- 需要确保 `GameManager` 在绑定设置面板前已经初始化 `settingPanelModel`。
- 如果 `GameManager.Instance.settingPanelModel` 为空，设置面板绑定会出问题。

### 设置面板每帧判断场景

`SettingPanelView.Update()` 根据当前场景名控制退出按钮显隐：
- 当前不是 `MainMenuScene` 时显示退出按钮。
- 当前是 `MainMenuScene` 时隐藏退出按钮。

影响：
- 功能直观，但每帧查询场景和设置按钮显隐可以后续优化为面板打开时刷新一次。
